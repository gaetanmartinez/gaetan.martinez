Mission pour l'ONCFM (Organisation nationale de lutte contre le faux-monnayage) : construire un modèle de prédiction capable de détecter automatiquement les faux billets en euros à partir de leurs caractéristiques physiques.

Contexte & besoin métier

L'ONCFM souhaite automatiser la détection des faux billets en euros. Aujourd'hui, le processus est manuel et chronophage. L'objectif est de fournir aux équipes une application capable, après scan d'un billet (longueur, hauteur, marges, diagonale), de prédire automatiquement s'il est authentique ou contrefait.

La mission est recommandée par l'agence européenne EMV (European Monetary Verification) et couvre 4 algorithmes imposés : K-means, Régression logistique, KNN et Random Forest.

Données

Un fichier CSV de 1 500 billets déjà scannés, avec 6 variables physiques mesurées :

length — Longueur du billet · Corrélation cible : +0,85 (variable la plus discriminante)
margin_low — Marge basse · Corrélation cible : −0,78
margin_up — Marge haute · Corrélation cible : −0,61
height_left / height_right — Hauteurs gauche et droite · Corrélations modérées
diagonal — Diagonale · Très faible corrélation avec la cible
NaN dans margin_low — Quelques valeurs manquantes imputées par la moyenne du train uniquement (pour éviter toute fuite de données)

Démarche

EDA — Analyse des corrélations avec la cible. length (+0,85) et margin_low (−0,78) ressortent comme variables clés.
Prétraitement — Split train/test 80/20 avec stratification (proportion vrai/faux préservée). Imputation des NaN sur le train uniquement. Standardisation StandardScaler.
Modélisation — 4 algorithmes testés avec validation croisée (5 folds, F1-score). GridSearchCV pour optimiser k dans KNN.
Évaluation — Comparaison sur Accuracy, Precision, Recall, F1-score et ROC AUC. Matrices de confusion pour tous les modèles.
Pipeline de production — Le meilleur modèle est sauvegardé dans un pipeline scikit-learn (Imputer + Scaler + Modèle) via joblib pour une utilisation en production.

Résultats & insights

Meilleur modèle : KNN — sélectionné automatiquement sur le F1-score. k optimal trouvé via GridSearchCV. Performances quasi-parfaites sur le jeu de test.
Variables clés : length (la plus discriminante), margin_low et margin_up permettent de distinguer clairement vrais et faux billets.
KMeans : moins performant car non supervisé — il ne connaît pas les labels pendant l'entraînement. Utile pour explorer mais pas pour prédire en production.
Pipeline de production : modèle sauvegardé avec joblib (Imputer + Scaler + KNN). Peut prédire directement sur un nouveau CSV brut, même avec des NaN.
Reproductibilité : random_state=42 partout, stratify sur le split, imputation calculée uniquement sur le train.

Stack & outils

Python 3.x scikit-learn pandas numpy matplotlib seaborn joblib Jupyter

Limites & prochaines pistes

Dataset limité — 1 500 billets. Les performances pourraient varier sur un dataset plus large et plus varié.
KMeans et les labels — L'inversion des clusters pour aligner les labels utilise y_test, ce qui est une légère fuite de données. Mentionné explicitement dans le notebook.
6 variables physiques uniquement — D'autres caractéristiques (couleur, texture, hologramme) pourraient enrichir le modèle.

Prochaines pistes : déployer le pipeline dans une API REST (FastAPI) · tester XGBoost ou LightGBM · ajouter une interface utilisateur simple pour les équipes ONCFM.
