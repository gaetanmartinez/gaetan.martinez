Contexte & besoin métier

Bottleneck est une boutique e-commerce spécialisée dans la vente de vins et spiritueux. L'entreprise gère son catalogue via deux systèmes non synchronisés : un ERP interne et une plateforme web. Un fichier de liaison assure la correspondance entre les deux.

Le directeur a commandé une analyse pour répondre à un besoin concret : comprendre les performances produits, identifier les risques de stock et de marge, et disposer d'une base fiable pour piloter les décisions commerciales.

Données

3 fichiers Excel fournis par le client :

erp.xlsx — 824 lignes · Prix, stocks, prix d'achat, statut
web.xlsx — Catalogue web · SKU, ventes, type produit
liaison.xlsx — Table de correspondance ERP ↔ Web

Qualité des données : 2 incohérences stock_status corrigées, prix négatifs traités, SKU non conformes supprimés. Dataset final : 712 lignes après fusion outer et nettoyage.

Démarche

Pipeline en 5 phases pilotées sur Trello :

Phase 1 — Cadrage : CDC, veille technologique (4 axes), parties prenantes
Phase 2 — Data Audit : nettoyage des 3 fichiers, rapports EDA automatisés (fg-data-profiling)
Phase 3 — EDA & Analyse : fusion, outliers IQR, CA/Pareto, stocks, marges, corrélations
Phase 4 — Validation : reproductibilité, narration Markdown, critères d'acceptation
Phase 5 — Restitution : documentation complète, dépôt final

Résultats & insights

CA total : 143 285,90 € calculé à partir de 712 articles
Loi de Pareto : 432 articles (60,7% du catalogue) génèrent 80% du CA — répartition plus homogène que la règle 20/80 classique
Outliers prix : 25 articles au-dessus du seuil IQR (84€) — tous justifiés (produits haut de gamme avec ventes confirmées)
Corrélation prix ↔ ventes : −0,58 — levier le plus net : le prix pèse directement sur les volumes
Corrélation stock ↔ ventes : +0,42 — les articles bien stockés tendent à mieux se vendre
Marge : 1 produit à marge négative identifié — perte théorique (total_sales = 0), à surveiller avant première vente

Limites & prochaines pistes

Données temporelles absentes — Pas d'historique daté. Impossible d'analyser les tendances ou la saisonnalité.
TVA 20% uniforme — Taux réel non fourni. L'hypothèse peut légèrement biaiser les calculs de marge.
Corrélations ≠ causalité — Les corrélations observées orientent la réflexion mais ne prouvent pas de lien de cause à effet.

Prochaines pistes : intégrer un historique daté pour une analyse temporelle · remplacer les merges pandas par dbt + DuckDB si le nombre de sources augmente · construire un dashboard interactif Streamlit pour la direction.
